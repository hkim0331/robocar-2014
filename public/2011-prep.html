ターミナル

  • ターミナルの文字の大きさは 14pt 以上とすること。

    ターミナルソフトウェアの種類はたくさんあり、 情報センターの場合、konsole がデフォルトです。

    konsole の文字の大きさを変えるには、

    1. konsole を立ち上げ、
    2. 設定 → フォント → 選択 メニューを選んで、

    3. 「サイズ」を 14 以上にする。 選んだフォントはどんなデザインか、 「The Quick いろはにほへと」の表示を見て確認しよう。

  • 本資料ではターミナルを黒字に白で表示します。 
    $ cal

    とあったら、 ターミナルで cal（エンター）とキー操作することを表します。 $ はコンピュータがターミナル上にあらかじめ表示する文字列 （プロンプトという）で、キーボードから打ってはいけません。

    プロンプト $ が表示された状態でないと、 コマンドを入力することはできない。

gedit

  • プログラムを作るエディタは gedit を使う。

    情報センターでは gedit 以外にも emacs や kwrite などのエディタ が使える。それらの使い方がわからなくても、 基本的に授業時間中のサポートはしません。

  • ターミナルと同じく、14ポイント以上の文字をつかうこと。

  • プログラミングしやすいフォントを選択すること。

    プログラムする人が読みやすければいいとは思うが、 フォントによっては lと1、0とO、qとgなどが判別しにくいものもある。 そうしたフォントを使ったためにエラーがみつけられない、 字下げがそろわない等は本末転倒です。

    ファニーなフォントでやりぬくという強い意志がない限り、 プログラミングではモノスペース （文字の種類によって幅が変化しない、等幅） のフォントを使う方が不必要な混乱を避けるためにもベターです。

gedit の起動は、ターミナルから gedit (エンターキー）。

$ gedit

こうして起動すると、起動した gedit を終了するまで、 ターミナルからコマンドを入力することはできない。

gedit の後ろに & をつけて gedit & で起動すると、 geditの終了をまたず、引き続き、 ターミナルで新しいコマンドを実行することができます。 

$ gedit &
$ date
2010年 4月16日 金曜日 18時33分06秒 JST
$

gedit をメニューに追加

ターミナルから gedit がめんどくさいなら、 メニューに gedit を追加しておくとよい。

  1. デスクトップの左下隅、K メニューを右クリック

  2. 現れるメニューからメニューを追加を左クリック
  3. エディタの項を右クリック

  4. 新規項目を左クリック
  5. 「項目名」に gedit

  6. コマンドに /usr/bin/gedit

  7. （お好みで）上の画面の右上方、 アイコンのエリアをクリックし、 お好みのアイコンを設定することができます。
  8. 変更をセーブする。
  9. gedit がメニューに現れる。

gedit プラグインのインストール

みんなの先輩は半角文字/全角文字に無頓着な人が多かった。 C のプログラムでは半角文字であるべきところを全角文字にしてしまうと、 エラーとなって動作しない。 とくに全角スペースは見つけにくいエラーのもとなので要注意。

ということで、gedit に 全角文字を入力したらそれを四角で囲んで表示する機能をプログラムしました。 以下の手順で有効化できます。

  1. まず、ターミナルから ds-install を実行。
    $ ds-install
  2. ds-install の実行後、gedit を立ち上げ、 
    $ gedit
  3. gedit の編集メニューから設定サブメニュー、プラグインタブを選び、 スクロールして空白文字の表示にチェックマークを入れる。
  4. 「プラグインの設定」をクリックして、 全角文字を囲む四角の色を選ぶことができます （下図の右上の赤四角のエリア）。

    図には Draw non-breaking spaces と Draw tabs のチェックも入っていますが、 全角文字だけをマークすればいいので、それらの機能は殺してあります。 下二つのチェックマークは無効です。だけど、いちばん上の Draw spaces はチェックを外すと全角マークも外れてしまいます。

以上の操作を正しく実行したのち、gedit を起動し直す。 全角文字が四角で囲まれていることがわかりますか（下図）。 上記「プラグインの設定」で赤を選んでいます。

こうすると、日本語で書いたコメントの部分や、プログラム中、エラーにならない日本語文字列も赤で囲まれるが、 ま、ノービスユーザなら、こっちのほうがいいっしょ。なれたらこの gedit プラグインを外せばいい。

iceweasel

  • iceweasel で *.c ファイルを開く[2010-04-26 追加]

    情報センターのブラウザ iceweasel に *.c ファイルを読んだら gedit でそれを開く設定をする。

    1. iceweasel を立ち上げ、 編集 メニューから設定を選ぶ。
    2. プログラムを選び、ファイルの種類から 「C ソースコード」を探す。
    3. 「C ソースコード」の「取り扱い方法」をクリック、 (毎回確認する、 あるいは GNU TexMacs Editor を使用が選ばれていたら変更する） 「場所」のところ、/usr/bin/gedit をタイプ。
  • iceweasel が開かない

    PCを強制終了したときに iceweasel がゾンビとなることがあるようです。

    ターミナルから、

    $ iceweasel_recover.sh

    で復活するそうです。（提供は情報処理センター）

ecc/jcc

昨年度授業担当者からの情報を分析し、 C コンパイラに小さな仕掛けを施したプログラムを作っています。 これについてはおいおい説明します。

ecc はターミナルから ecc （エンターキー）で起動します。

$ ecc hello_world.c
Hello, World!
$

ecc を強制終了するにはコントロール C (コントロールキーを押したまま、C）のキー操作をします。

作業ディレクトリ

授業で作るプログラムはディレクトリ c の中にセーブするようにする。 学期の最初に一度だけ、mkdir してそのディレクトリを作っておく。 

$ mkdir ~/c

gedit でプログラムを作成したらそのディレクトリ c の中にセーブ、 ディレクトリ c に移動してコンパイル、実行すると吉。

$ cd ~/c
$ cc sample.c
$ ./a.out
