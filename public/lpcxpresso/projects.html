複数のプロジェクト

robocar.zip を展開したプロジェクトで開発を進めているとする。

課題をひとつクリアして、別の課題に取り組もうとしたとする。

main.c を上書きしてしまうと、クリアした過去の課題が残らない。 main.c をプロジェクト内の別ファイルにリネームしても、 古い main() をコメントアウトするか別の関数にリネームしない限り、 新しいmain.c 中の main() とバッティングし、 ビルド時にエラーになる。

簡単な回避策は二つ。

1, main.c をプロジェクト外部の別ファイルにコピー

  1. LPCXpresso の main.c の上で右クリック、
  2. コピーを選んで、
  3. ファイルブラウザ上の適当な場所にペースト。
  4. ペーストした main.c を適当な名前に変えておく。
  5. プロジェクトに戻って、main.c を新しい内容で書き換える。

昔のプロジェクトに戻りたくなったら、セーブしたファイルを main.c にコピーペーストで戻せばよい。

ひとつのプロジェクトを使い回す作戦。 ファイルの容量は食わないが、 プロジェクトを切り替える度のコピーペーストはめんどうではないか？

2, 別のプロジェクトを立ち上げる

main.c のコピーペーストよりも、別プロジェクトを立ち上げる方が便利。 LPCXpresso は複数のプロジェクトを管理できるようにできている。

  1. LPCXpresso で robocar プロジェクトを選択、
  2. File メニューから Rename... を選んでプロジェクトを別の名前に。
  3. robocar.zip を新たに import する。 下図は古いプロジェクトを eight と名称替えし、 新たに robocar.zip を import して、src/main.c を開いたところ。

昔の課題を手直ししたくなったら、 そのプロジェクト（上の例では eight）を開けば良い。
